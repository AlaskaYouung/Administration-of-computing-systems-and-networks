# Лабораторные работы №1-6
Во всех шести работах моделируется корпоративная вычислительная сеть в Cisco Packet Tracer. Используются статические и динамические протоколы маршрутизации,
VLAN (косвенно), коммутаторы, маршрутизаторы, а также беспроводные точки доступа. Основной упор — на адресацию (IPv4/IPv6),проверку связности (ping, traceroute),
таблицы маршрутизации и MAC-таблицы.

Кратко по работам:
- ЛР №1 – Базовая топология из двух сетей (10.1.0.0/16, 192.168.1.0/24, 192.168.99.0/30). Настройка IP на ПК, проверка связности.
- ЛР №2 – Расширенная сеть с 6 ПК в одной подсети. Использование ifconfig, ping, traceroute.
- ЛР №3 – Одна подсеть 192.168.1.0/24, 6 ПК, два коммутатора. Анализ MAC-таблиц (mactable).
- ЛР №4 – Два маршрутизатора (R1, R2), статическая маршрутизация. Таблицы маршрутизации, ping, traceroute между подсетями.
- ЛР №5 – Маршрутизатор R1, два коммутатора (S1, S2). Просмотр ARP-таблиц и MAC-таблиц.
- ЛР №6 – Беспроводная сеть: 2 точки доступа (AP1, AP2), 4 лэптопа (LP1–LP4). Настройка Wi-Fi, iwconfig, ping, traceroute.

# Контрольная работа в файле АДМ Зубенко
Цель:
Спроектировать и адресовать корпоративную сеть с ядром из 4 маршрутизаторов (R0–R3), 4 подсетями рабочих станций (по 6 ПК в каждой), магистральными подсетями, а также двумя беспроводными точками доступа.

Что сделано:
- Выделены подсети пользователей (192.168.1.0/28 для IPv4, fc00::/96 для IPv6).
- Построен граф ядра и полной сети.
- Описаны технические средства (Cisco 3600, Catalyst 1200, WAP571).
- Разработаны планы адресации IPv4/IPv6 для подсетей пользователей и магистрали.
- Составлены таблицы маршрутизации для R0–R3 и для каждой подсети.
- Добавлены 2 точки доступа (AP1 — корневая, AP2 — повторитель) с SSID Subnet5, шифрованием WPA2/WPA3.

Результат:
Полностью спроектированная сеть с поддержкой IPv4/IPv6, статической маршрутизацией, беспроводным сегментом и резервными адресными пространствами.
